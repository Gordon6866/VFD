### Bug修复：Turbopack 崩溃
- **根因**：Git Bash 中 `2>nul` 被误解析为创建名为 `nul` 的文件（而非 Windows NUL 设备）
- **现象**：Turbopack 编译 CSS 时读取 `nul` 文件导致 panic 崩溃 (500)
- **修复**：删除 `nul` 幽灵文件 + 清理 `.next` 缓存 + 重启服务器
- **教训**：在 Git Bash 中避免使用 `2>nul`，应使用 `2>/dev/null`
